# 15장 AWS와 GCP로 배포하기

- 다른 사람이 이용하는 서비스이므로 개발 환경과 동일하게 진행할 수 없음. 서버 유지와 에러 내역 관리, 보안 관리 등이 필요함.
- 사전 작업 마무리 후 아래의 플랫폼으로 서비스를 배포할 수 있음.
- aws: 아마존 웹 서비스
- gcp: 구글 클라우드 플랫폼


- morgan과 express-session
- process.env.NODE_ENV: 배포 환경인지 개발 환경인지 판단할 수 있는 환경 변수
- 시퀄라이즈: 비밀번호가 하드코딩 되어 있음. json 파일은 변수를 사용할 수 없음.
- cross-env 패키지: 동적으로 환경 변수를 변경할 수 있음.
- xss 공격 방어: 악성 사용자가 사이트에 스크립트를 삽입하여 하는 공격. csurf 패키지로 방어 가능
- pm2: 원활한 서버 운영을 위한 패키지. 에러로 서버가 꺼지면 다시 켜줌
- winston: console.log을 대체하는 모듈. 콘솔 로그는 서버가 종료되면 로그들도 사라져버림. 윈스턴은 로그를 다른 곳에 저장해줌
